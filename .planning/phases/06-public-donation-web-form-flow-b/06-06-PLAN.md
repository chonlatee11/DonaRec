<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| donor browser -> Next public route | unauthenticated multipart submission |
| Next route -> Go API | server-side proxy without a session |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-06-21 | Spoofing | reusing session-bound BFF for public route | high | mitigate | New session-less passthrough — never calls getServerSession/getBffToken (would 401 every donor); no Authorization header sent |
| T-06-22 | Tampering | client-trusted validation only | medium | mitigate | Client gating is UX only; Go re-validates fields (validator), slip (magic-byte), captcha (siteverify) server-side — client is never authoritative |
| T-06-23 | Repudiation | reference/receipt confusion | high | mitigate | Confirmation copy states "not yet a receipt"; reference number uses a distinct typeface/theme from receipt numbers (D-84) |
| T-06-24 | Information Disclosure | API base URL exposure | low | mitigate | API_BASE_URL stays server-side in the Next route; browser never learns the Go origin (no CORS introduced) |
| T-06-SC | Tampering | npm install | low | accept | Turnstile dep already added + verified in plan 05 |
</threat_model>
