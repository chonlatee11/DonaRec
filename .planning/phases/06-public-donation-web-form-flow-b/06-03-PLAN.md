<objective>
The load-bearing vertical slice: a donor can submit a complete donation request over a real, unauthenticated HTTP path and it lands atomically in `pending_review` as `source=flow_b`, with the tax ID encrypted, consent recorded, slip stored and referenced, the submission audited under the seeded public-web actor, and an `ack_email` outbox job enqueued — never a partial/orphan record (D-76/78/79/80/81, FR-01/02/03/04).

Per CLAUDE.md Conventions Integration-test gate, this seam is NOT done without an E2E test over the real HTTP path — this phase adds the project's first unauthenticated route group, exactly the class the gate exists for.

Purpose: This is the whole point of Flow B — everything else (frontend form, ack worker, queue) is a wrapper around this atomic path.
Output: `CreatePublicSubmission` service method, `public_handler.go`, the `/api/public` route group wired with rate-limit+captcha middleware, and `e2e_public_test.go` proving the full path.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| donor browser -> /api/public/donations | untrusted unauthenticated multipart POST (fields + file + captcha token) |
| handler -> MinIO/Postgres | validated input crosses into storage + encrypted DB writes |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-06-09 | Tampering / EoP | slip file-type spoofing | high | mitigate | storage.PutSlip magic-byte detection (mimetype.Detect) + size cap BEFORE any DB write (Pitfall 4); extension/Content-Type never trusted |
| T-06-10 | Information Disclosure | donor tax ID at rest | high | mitigate | crypto.EncryptField AES-256-GCM envelope, identical to Flow A; plaintext never reaches Postgres |
| T-06-11 | Elevation of Privilege | unauthenticated route reachable mutations | high | mitigate | publicGroup registers ONLY POST /donations (create one pending_review); no update/delete/reveal/approve exposed (ASVS V4 code-review assertion) |
| T-06-12 | Repudiation | public submit without an actor | high | mitigate | AppendAuditEntryTx in-tx with the fixed-UUID public-web actor (Pitfall 1) |
| T-06-13 | Data integrity | partial submission (row w/o slip) | high | mitigate | Single WithTx (create+submit+slip+audit+outbox); slip PUT + validation fail-fast before tx; E2E asserts no orphan row |
| T-06-14 | Information Disclosure | verbose enumeration errors | medium | mitigate | Handler returns generic validation errors, never differential responses (no donor-master/dedup exists yet — D-43) |
