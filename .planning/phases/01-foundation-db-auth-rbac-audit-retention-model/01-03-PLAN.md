<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| App → PostgreSQL (PII fields) | Only ciphertext crosses to DB; plaintext never persisted (PDPA) |
| KEK source (env/secrets) → app | KEK loaded at startup; never in source or DB |
| Role → PII visibility | Maker sees mask; Admin/Checker may reveal (gated + audited) |
| App/DBA → legal-hold record | Neither can hard-delete under legal_hold |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-pii-01 | Information disclosure | PII at rest | mitigate | AES-256-GCM envelope; DB stores ciphertext+wrappedDEK only; round-trip + tamper tests (NFR-02, D-25) |
| T-1-pii-02 | Information disclosure | KEK leakage | mitigate | KEK from DONAREC_KEK env only; constructor errors if absent; no key literal in source (Pitfall 5) |
| T-1-pii-mask | Information disclosure | over-broad PII visibility | mitigate | mask-by-default last-4; reveal gated to Admin/Checker (D-10/D-11) + just-in-time (D-12); reveal audited via 01-02 path (D-13) |
| T-1-pii-03 | Tampering | ciphertext tamper | mitigate | GCM authenticated encryption detects modification on Decrypt |
| T-1-retention-01 | Tampering/Destruction | hard-delete under legal_hold | mitigate | app GuardHardDelete + DB BEFORE DELETE trigger (defense-in-depth, D-19) |
| T-1-retention-02 | Repudiation | wrong/absent retain_until | accept | config-driven default (D-18); exact PDPA period pending DPO (stakeholder gate, non-blocking) — schema accepts any value |
| T-1-pii-04 | Information disclosure | blind index reverses PII | accept | HMAC-SHA256 with separate index key; not reversible; full search usage deferred to Phase 3 |
</threat_model>
