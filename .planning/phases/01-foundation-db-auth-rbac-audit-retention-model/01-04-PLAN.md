แก้ donnarec-api/docker-compose.yml สามจุด (อย่าแตะ service/key อื่น):

GAP 1 — ครอบค่า DONAREC_KEK ด้วย double quotes (ปัจจุบันบรรทัด ~102):
- เดิม: `DONAREC_KEK: ${DONAREC_KEK:?DONAREC_KEK is required — generate with: openssl rand -hex 32}`
- ใหม่: `DONAREC_KEK: "${DONAREC_KEK:?DONAREC_KEK is required — generate with: openssl rand -hex 32}"`
- เหตุผล: ข้อความ default error มี ': ' (โคลอน+เว้นวรรค) → unquoted scalar ถูก YAML ตีความเป็น mapping → ทั้งไฟล์ parse fail. การครอบ double quote ทำให้เป็น string scalar เดียว

GAP 2 — แก้ path ของ postgres init-script volume (ปัจจุบันบรรทัด ~35):
- เดิม: `- ./scripts/create-multiple-dbs.sh:/docker-entrypoint-initdb.d/create-multiple-dbs.sh:ro`
- ใหม่: `- ../scripts/create-multiple-dbs.sh:/docker-entrypoint-initdb.d/create-multiple-dbs.sh:ro`
- เหตุผล: ไฟล์จริงอยู่ที่ repo-root scripts/ (compose อยู่ donnarec-api/ → ต้องขึ้นไปหนึ่งชั้น) เทียบกับ keycloak volume ที่ใช้ ../keycloak ถูกอยู่แล้ว
- แก้ usage comment ให้สอดคล้อง (ปัจจุบันบรรทัด ~12): `#   4. Run: bash scripts/seed-admin.sh  (creates first admin user)` → `#   4. Run: bash ../scripts/seed-admin.sh  (creates first admin user)`

GAP 5 — แก้ healthcheck ของ keycloak service ให้ใช้กับ Keycloak 26 (ปัจจุบันบรรทัด ~81-86) + เพิ่ม env:
- เพิ่มใน keycloak `environment:` (ใส่ใกล้ KC_HTTP_ENABLED): `KC_HEALTH_ENABLED: "true"` — เปิด management listener บนพอร์ต 9000 (จำเป็น ไม่งั้น 9000 refused)
- แทนที่ healthcheck `test:` เดิม `["CMD-SHELL", "curl -f http://localhost:8080/health/ready || exit 1"]` ด้วย curl-less probe ยิงพอร์ต 9000 ผ่าน /dev/tcp ของ /bin/sh (image รองรับ — verified):
  `test: ["CMD-SHELL", "exec 3<>/dev/tcp/localhost/9000 && printf 'GET /health/ready HTTP/1.1\r\nHost: localhost\r\nConnection: close\r\n\r\n' >&3 && grep -q '200' <&3"]`
  คง interval/timeout/retries/start_period เดิมไว้ (start_period: 60s สำคัญเพราะ Keycloak boot ช้า)
- เหตุผล: Keycloak 26 image ไม่มี curl (NO_CURL) และเสิร์ฟ /health บน management port 9000 ไม่ใช่ 8080 (8080 → 404)

หมายเหตุ: ใช้ Edit ราย hunk; อย่าเขียนทับทั้งไฟล์ ห้ามแตะ DATABASE_URL sslmode=disable (เป็น local-dev design decision)
