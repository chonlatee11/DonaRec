<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser/Next.js → Go API | Untrusted Bearer JWT crosses here; must be validated against Keycloak JWKS |
| Go API → Keycloak | Token issuer; API trusts only tokens signed by configured realm |
| Go API → PostgreSQL | Parameterized queries only; app role privileges constrained |
| Operator → Keycloak realm config | Password policy / lockout are data, must be applied via realm import |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-auth-01 | Spoofing | auth/middleware.go token validation | mitigate | go-oidc Verifier enforces signature + `iss` (realm URL) + `aud` (donnarec-backend); reject 401 otherwise. Test wrong-aud/wrong-iss → 401 (Pitfall 3) |
| T-1-auth-02 | Spoofing | Keycloak login | mitigate | bruteForceProtected + maxLoginFailures=5 in realm-donnarec.json (D-07); password policy length(8)+digits (D-06) |
| T-1-auth-03 | Spoofing | session lifetime | mitigate | accessTokenLifespan 300s + ssoSessionIdleTimeout 1800s + maxLifespan 28800s (D-08) in realm config |
| T-1-rbac-01 | Elevation of privilege | rbac.go RequireRoles | mitigate | server-side guard on every protected route; missing role → 403; verified by TestRequireRoles. Multi-role evaluated per-role (D-02) |
