## Phase Goal

**As a** Checker or Admin, **I want to** download an access-controlled Excel/CSV export of issued donations mapped to e-Donation fields (full 13-digit ID, donation date, cash type), with every export audited and download-logged, **so that** I can manually key the data into the Revenue Department's e-Donation system correctly and accountably.

<objective>
Implement the e-Donation export endpoint end to end: an audited, RBAC-gated, stream-only export of issued donations to .xlsx (primary) and BOM CSV (secondary), with columns driven by the edonation_config field mapping (D-75) and full national IDs decrypted through Phase 3's audited-reveal discipline.

Purpose: FR-30 / SC#1 — staff can generate the file they need for manual e-Donation keying, with least-privilege access and a durable audit trail.
Output: `internal/edonation` service/handler/xlsx/csv; export route wired in main.go under RequireAnyRole(Checker,Admin); E2E subtests in cmd/server/e2e_test.go proving the real HTTP path.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser/BFF → Go API | Bearer token; RequireAnyRole(Checker,Admin) gate is the real authority for export |
| Go service → encrypted PII | Ciphertext decrypted only inside the audited tx; plaintext exists only in-memory for the response body |
| Go service → HTTP response body | File streamed in memory; never persisted to disk/MinIO (D-74) |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-05-02-RBAC | Elevation of Privilege | /api/edonation/export guard | high | mitigate | RequireAnyRole(Checker,Admin) OR-guard (not RequireRoles AND); Task 3 E2E asserts Maker 403, Checker/Admin 200 |
| T-05-02-PERSIST | Information Disclosure | export file handling | high | mitigate | Stream-only via exportfile.StreamXLSX/CSV to ResponseWriter; no os.Create/TempFile; E2E asserts buckets unchanged (D-74) |
| T-05-02-UNAUDITED | Repudiation | audited decrypt | high | mitigate | One AppendAuditEntryTx summary row (actor/range/count) committed before plaintext returns; tx rolls back if audit fails (D-64) |
| T-05-02-BULK | Information Disclosure | bulk PII exfiltration by compromised Checker/Admin | medium | accept | D-64 audit gives after-the-fact detectability (who/when/range/count); hard record-cap not CONTEXT-locked — accepted for MVP |
| T-05-02-LOGPII | Information Disclosure | handler/service logs | medium | mitigate | Log donation count only, never plaintext national IDs (V7, mirrors RevealPII logging discipline) |
| T-05-02-SQLI | Tampering | export filter params | medium | mitigate | sqlc.narg parameterized SearchIssuedForExport; format param allowlist-validated |
</threat_model>
