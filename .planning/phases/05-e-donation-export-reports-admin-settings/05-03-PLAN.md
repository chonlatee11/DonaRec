## Phase Goal

**As a** hospital administrator responsible for donor records, **I want to** have scheduled backups that cover both the database and object storage, plus a proven, evidence-backed restore procedure, **so that** if the server is lost I can recover every donation record, slip, and frozen receipt PDF completely — not merely hope a cron job worked.

<objective>
Deliver verified backup and restore for NFR-08: docker-compose companion services that dump Postgres (custom format) and mirror both MinIO buckets on a schedule with retention, a restore-proof Go integration test that actually restores real artifacts into fresh testcontainers Postgres + MinIO and asserts data completeness, and a runbook capturing the procedure and recorded evidence.

Purpose: SC#4 — a backup runs regularly AND a documented restore has been performed successfully (restore verified, not just configured).
Output: `docker/backup.Dockerfile`, backup/restore scripts, compose companion services, `internal/backupverify/restore_test.go`, `docs/BACKUP_RESTORE_RUNBOOK.md`.

> This plan has zero application-runtime code and zero UI footprint — it is infra/ops beside the Go binary. It runs in Wave 1 parallel with 05-01 (no shared files).
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Postgres/MinIO → backup volume | Dump + mirrored objects contain ciphertext PII and KEK-adjacent data; volume must be access-restricted and never in VCS |
| backup container → restore target | Restore into a fresh environment must preserve data completeness/integrity |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-05-03-DUMPLEAK | Information Disclosure | backup dump artifacts | high | mitigate | Backups volume/path gitignored; never committed to VCS; documented as sensitive (ciphertext + KEK-adjacent) in runbook |
| T-05-03-INCOMPLETE | Denial of Service (data loss) | DB-only backup missing object storage | high | mitigate | Backup covers BOTH pg_dump AND mc mirror of slips+receipts (D-72); restore-proof asserts object presence |
| T-05-03-BADFORMAT | Denial of Service | pg_dump plain format blocks restore | medium | mitigate | Enforce -Fc custom format (Pitfall 4); restore-proof test would fail if wrong format used |
| T-05-03-UNVERIFIED | Repudiation | "configured but never restored" | high | mitigate | Real restore-proof integration test + recorded evidence in runbook (D-73), not a cron-config check |
</threat_model>
