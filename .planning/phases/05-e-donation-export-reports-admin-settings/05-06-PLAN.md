## Phase Goal

**As a** Checker or Admin, **I want to** use a dedicated e-Donation screen to export the file I need and track which receipts are still unkeyed against the deadline, **so that** I can complete manual e-Donation keying accurately and never miss the monthly cutoff — all from one workflow surface.

Build Screen 7 (e-Donation Export + Keyed-Status/Aging) end to end on the existing Next.js back-office: BFF proxy routes, the Export tab (filter + count preview + PII-warning confirm + streamed download), the Aging tab (bucket stat cards + selectable table + bulk action bar), the two new badge components, the new checkbox primitive, and the Checker/Admin-gated nav item.

Output: `/e-donation` route + components + BFF routes + nav item + i18n keys.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Next.js BFF | Same-origin fetch; token obtained server-side, never exposed to browser |
| BFF → Go API | Server-side Bearer forward; Go RequireAnyRole is the real authority |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-05-06-TOKEN | Information Disclosure | access token exposure | high | mitigate | BFF forwards Bearer server-side (bffForward/getBffToken); token never reaches browser (existing D-R1 pattern) |
| T-05-06-UXGATE | Elevation of Privilege | nav-link role gating | medium | mitigate | isCheckerOrAdminViewer is a UX hint only; Go RequireAnyRole(Checker,Admin) on /api/edonation/* is the authority (05-02/05-04) |
| T-05-06-FILECUSTODY | Information Disclosure | downloaded PII file custody | medium | mitigate | ExportConfirmDialog amber warning instructs secure handling/deletion (D-64); server persists nothing (D-74) |
| T-05-06-BINARY | Tampering | export binary passthrough | low | mitigate | BFF passes the xlsx/csv body through unchanged (no re-encode); Content-Disposition preserved |
</threat_model>
