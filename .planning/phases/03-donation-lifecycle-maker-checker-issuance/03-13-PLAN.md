<objective>
Migrate the remaining write vertical — create/edit (Screen 2), slip upload (Screen 3 section), and the
cancel/void-and-reissue mutations (Screen 3 action panel) — to the D-R1 BFF + TanStack pattern. This closes the
last FE↔BE contract gap: the create/update REQUEST bodies use FE field names (`national_id`, `address`, `email`,
`note`) that do NOT match the Go `CreateDonationRequest`/`UpdateDraftRequest` (`donor_tax_id`, `donor_address`,
`donor_email`, `notes`) — the BFF create/update routes map them. Add BFF routes for create/update/submit/cancel/
reissue/slip, convert the form + dialogs to TanStack mutations, and extend the E2E to cover create + cancel over
the real router.

Purpose: Maker create→edit→slip→submit and Checker cancel/void-reissue all work end-to-end against the real API.

<threat_model>
## Trust Boundaries
| Boundary | Description |
|----------|-------------|
| browser → BFF create/update/slip/cancel | untrusted write payloads + file upload; token stays server-side |
| BFF → Go | Bearer forward; Go re-validates body (validator), magic-byte slip, RBAC, SoD, edonation_keyed guard |

## STRIDE Threat Register
| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-01 | Tampering | slip upload via BFF | mitigate | Go performs magic-byte + 10MB validation (authority); BFF streams bytes without trusting extension/Content-Type |
| T-13-02 | Information Disclosure | create/update PII (national ID) | mitigate | plaintext national ID travels browser→BFF→Go over TLS; Go encrypts at rest (AES-GCM); no PII logged; token never in browser |
| T-13-03 | Elevation of Privilege | cancel/reissue via BFF | mitigate | Go checker/admin route guard + edonation_keyed rd_confirmation guard (D-51) re-enforced server-side; BFF passes 403/409 through |
| T-13-04 | Repudiation | cancel/void-reissue | mitigate | Go writes audit_log in the same tx (FR-19); UI cannot bypass |
| T-13-SC | Tampering | npm | accept | no new package installs (TanStack added in 03-10) |
</threat_model>
