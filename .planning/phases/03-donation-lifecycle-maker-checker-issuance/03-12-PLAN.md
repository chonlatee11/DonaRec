<threat_model>
## Trust Boundaries
| Boundary | Description |
|----------|-------------|
| browser → BFF [id]/[id]/pii/[id]/approve… | untrusted; token must stay server-side; reveal must be audited by Go |
| BFF → Go | server-side Bearer forward; Go re-enforces RBAC + SoD + reveal audit |

## STRIDE Threat Register
| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-01 | Information Disclosure | [id]/pii BFF route | mitigate | reveal forwarded to Go (audited, checker/admin gate); plaintext returned only for authorized roles; session-only client state, not persisted |
| T-12-02 | Elevation of Privilege | approve/return/reject via BFF | mitigate | Go re-enforces RequireAnyRole + service SoD; UI flags are hints; BFF passes Go 403/409 through |
| T-12-03 | Information Disclosure | token handling | mitigate | getServerSession server-side; BFF responses never carry the access token (D-R1) |
| T-12-04 | Tampering | slip_url composition | mitigate | presigned URL fetched server-side via Go /:id/slip; short TTL owned by Go (T-03-16) |
| T-12-SC | Tampering | npm (vitest devDependency) | mitigate | blocking-human legitimacy checkpoint (verify vitest on npmjs.com) before install; single well-known devDep; not shipped to the browser bundle |
</threat_model>
