<objective>
Fix the trigger bug (#5) at its source: change the donation list contract so `GET /api/donations`
returns the D-R2 pagination envelope `{"data":{"items":[...],"total":N,"page":P,"per_page":20}}`
instead of the bare `{"data":[array]}` that crashes `DonationTable` on `undefined.length`.

Backend adds a real COUNT (same filter predicate as the page query) for `total`, enriches each
list row with the creator display name (`created_by`) and `created_by_id` (users.id UUID), and the
E2E test is extended to assert the new envelope over the production router with a real Keycloak token.

Purpose: this is the backend half of the list vertical slice. 03-10 migrates the list UI to consume it.
Output: aligned list contract + COUNT + creator-name join + E2E assertion (integration gate for the list flow).
</objective>
