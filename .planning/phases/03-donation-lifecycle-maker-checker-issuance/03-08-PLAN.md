## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client upload → server | file bytes (server magic-byte validation is authority) |
| client reveal request → server | PII decryption gated + audited server-side |
| destructive dialogs → server | cancel/reissue guarded by server role + keyed checks |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-34 | Information Disclosure | revealed PII persists/leaks client-side | mitigate | reveal is session-only (client memory); reload re-masks; server audits every reveal (03-06) |
| T-03-35 | Tampering | client bypasses file validation | mitigate | client size pre-check is UX only; server magic-byte allowlist (03-04) is authority; 422/413 surfaced |
| T-03-36 | Repudiation | keyed receipt cancelled without RD confirmation | mitigate | CancelDialog forces RD-confirmation textarea when keyed; server re-checks (ErrEDonationKeyedConfirmation) |
| T-03-37 | Information Disclosure | consent text version mismatch | accept | consent_text_version shown from config + recorded server-side (03-03); UI displays the active version |
</threat_model>
