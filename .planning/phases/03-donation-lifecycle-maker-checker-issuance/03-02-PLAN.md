## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → Go API | bearer token crosses; token must come from authenticated Keycloak session, never hardcoded |
| third-party registries → build | shadcn blocks pulled at build; only official registry permitted |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-05 | Spoofing | API client auth | mitigate | bearer token sourced from Keycloak OIDC session in lib/auth.ts; no static/hardcoded token; env for base URL only |
| T-03-06 | Tampering | supply chain via shadcn registry | mitigate | official shadcn registry only (UI-SPEC Registry Safety); no third-party registry added |
| T-03-07 | Information Disclosure | PII rendered client-side | accept | this plan renders no donor data; masked-by-default enforced server-side (03-03) and in MaskedIdField (03-08) |
