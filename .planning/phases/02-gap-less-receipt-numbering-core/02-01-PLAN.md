<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| app (donnarec_app role) → PostgreSQL | App issues counter/ledger writes; DB constraints are the un-bypassable backstop |
| migration (superuser) → runtime (donnarec_app) | DDL + GRANT/REVOKE define what the runtime role may do |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | receipt_numbers ledger rows | mitigate | `REVOKE UPDATE, DELETE ON receipt_numbers FROM donnarec_app` — issued numbers immutable at DB level (Task 1) |
| T-02-02 | Tampering | duplicate (fiscal_year, running_no) | mitigate | `UNIQUE (fiscal_year, running_no)` DB constraint as backstop independent of app logic (Task 1) |
| T-02-03 | Tampering | SQL injection via dynamic SQL | mitigate | sqlc named params only (`@fiscal_year`, `@formatted`); no string concatenation (Task 2) |
| T-02-04 | Elevation of Privilege | running_no sourced from a SEQUENCE (gap on rollback) | mitigate | running_no is plain INT fed by counter table; SERIAL/nextval forbidden (Task 1 acceptance) |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | No new packages introduced; all deps already in go.mod/go.sum (per 02-RESEARCH Package Legitimacy Audit) |
</threat_model>
