<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| caller (Phase 3 issuance tx) → Allocate | Caller owns the tx; Allocate must operate inside it and bubble errors so caller rollback undoes the number |
| Allocate → PostgreSQL counter/ledger | Row lock + UNIQUE constraint enforce the gap-less invariant below the app |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-07 | Tampering | gap created by allocating in a separate tx | mitigate | Allocate takes pgx.Tx (never a Pool); rollback of caller tx undoes counter+ledger together (D-33, Pitfall 2) |
| T-02-08 | Tampering | duplicate number via read-max+1 | mitigate | Allocate uses counter FOR UPDATE + increment; no MAX query (acceptance grep) (Pitfall 3) |
| T-02-09 | Repudiation | number reserved on a draft then abandoned | mitigate | Number is born only at ledger INSERT inside the issuance tx; no pre-compute/reserve path exists (D-35) |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | No new packages; pgx/sqlc/testcontainers already in go.mod |
</threat_model>
