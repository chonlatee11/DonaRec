<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| outbox_jobs → worker | job payload references a donation whose snapshot drives render/email |
| worker → MinIO/EmailSender | PDF bytes + donor email leave the process |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-04-11 | Tampering (data integrity) | concurrent job claim | medium | mitigate | Atomic UPDATE...FOR UPDATE SKIP LOCKED claim (RESEARCH Pattern 1) — no double-processing |
| T-04-12 | Denial of Service | infinite auto-retry on permanent failure | medium | mitigate | attempts < max_attempts guard + next_attempt_at backoff → terminal 'failed' dead-letter (D-57); staff manual resend/download always available |
| T-04-13 | Information Disclosure | PII/attachment in logs | medium | mitigate | Pattern C logging (operation + donation_id only); email body/PDF/tax-id never logged |
| T-04-14 | Tampering | re-render mutating an issued receipt | high | mitigate | Skip render when receipt_pdf_object_key already set (D-56 freeze) — verified in worker_test.go |
</threat_model>
