<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| admin template HTML → render engine | admin-authored markup is trusted-but-risky (XSS/SSRF axis) and executed by a browser engine |
| donor field data → template | donor name/address is untrusted end-user input substituted into admin HTML |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-04-07 | Tampering / Information Disclosure | inline `<script>` in admin template | high | mitigate | emulation.SetScriptExecutionDisabled(true) — verified by render_sandbox_security_test.go |
| T-04-08 | Information Disclosure / SSRF | external/internal resource fetch during render | high | mitigate | fetch.Enable + FailRequest-all AND page.SetDocumentContent (no URL navigation) AND network-isolated sidecar (04-02) — three layers |
| T-04-09 | Tampering | donor field breaking out of HTML structure | high | mitigate | html/template contextual autoescaping; never trusted-raw-HTML-wrap the whole template |
| T-04-10 | Tampering | render non-determinism hiding a tofu/font regression | medium | mitigate | exact-byte golden PNG on pinned Chromium digest + bundled font (SC#2 load-bearing) |
</threat_model>
